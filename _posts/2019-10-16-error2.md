## Javascript 유효성 검사 오류

<br/>
<br/>
<br/>


나는 요즘 Javascript를 본격적으로 배우기 위해 Nomad Coders 의 To Do List 프로젝트를 수강 중이다.

​

블로그에서 Javascript 문법을 포스팅하는 것과 실제로 응용하는 것은 천지 차이였다 ;;;

코드를 읽고 분석할 수는 있지만 백지에서 과연 Javascript로 원하는 것을 도출해 낼 수 있을지는 아직까지 의문이다.

하지만 하다보면 늘 것이라고 믿고 있다.

​

아무튼, 나는 수강 중, 문제점을 마주했다.

분명 Nico가 진행하는 대로 똑같이 진행하였음에도 불구하고

​
​![](https://doozi316.github.io/assets/images/error2/1.png)


위의 스크린 샷과 같이 refresh를 하면 undefined가 출력되는 현상을 겪게 되었다.

(Nico는 정상작동하던데 난 왜 ... ㅠㅠ Nico의 code를 pull해와서 사용해봤지만 결과는 같았다.)

​

그래서 나는 구글링으로 undefined에 대한 유효성 검사를 해야한다는 것을 알게 되었다.

​

유효성 검사란?

undefined, null, "" 등 원치 않는 값이 실행될 경우 false를 return 하게끔 하는 로직이다.

​

나는 기존의 객체 toDoObj의 text가 undefined로 출력되지 않길 원했다.

```
const toDoObj = {
        text : text,
        id : newId  
    };
```
따라서 아래와 같은 코드를 객체 뒤에 추가해주었다.

```
if (typeof toDoObj.text === "undefined" || toDoObj.text === null|| toDoObj.text === ""){
        return false;
    }
```
It works!

처음처럼 refresh 할 때마다 undefined 리스트가 출력되진 않게되었지만

가장 처음 refresh를 하면 반드시 undefined가 하나는 출력된다는 것이다.

​

정말 의문인게, 객체 안엔 return false 때문인지 아무것도 저장되지 않았다.

그런데 출력은 왜 되는거야???

​​
​![](https://doozi316.github.io/assets/images/error2/2.png)


배열에 객체 하나만 존재하는게 보인는가?? 근데 undefined는 왜 출력되고 있는거야 ㅠㅠㅠㅠ
