진짜 나흘동안 온갖 사이트를 구글링했었다.

나는 header의 content-type을 잘 못 지정해주어서 에러가 발생하는 줄 알았다.

하지만 그게 아니었고, 결국 고심 끝에 질문을 직접 등록하게 되었다.

나는 왜인진 모르겠지만 질문을 올리는 일이 굉장히 쑥스럽게? 부끄럽게? 느껴져서 정말정말 고심하다가 이건 아니다 싶을때 올리곤 한다. 이번엔 유난히 오랜 시간이 걸렸고.(4일...) 

일주일의 절반을 에러를 찾는 일에만 몰두하다보니 자존감도 많이 낮아지고 많이 우울했다. ㅋㅋ...

하지만 에러를 해결함으로써 극복되었다.ㅋㅋㅋ

여기서 얻은 교훈은... 안풀릴 땐 오랜 시간동안 혼자 전전긍긍하지 말고 어디든 질문을 던지자는 것이었다.

아무런 발전 없이 자책하는 일을 반복하게 되면 개발에도 진척이 없고 멘탈관리도 안되더라.

​

아무튼 에러의 원인은, formData로 parameter을 보낼 때, key, value 값을 함께 보내지 않는가?

난 여기 key 값을 프론트와 백엔드 쪽에서 동일하게 적용하지 않아서 'file'이라는 key 값을 찾을 수 없다고 에러가 뜬 것이었다.

(프론트에선 key를 'fileList' 정의했고, 백엔드 쪽에선 'file'이라고 정의를 했었다. ㅠ)

너무 허무해서 에러를 해결했음에도 똥 안닦은 기분이었다.

진짜 바보같은 실수를...ㅋㅋㅋ 

영문 몇글자가 사람을 이렇게 미치게 만든다 ㅋㅋㅋ

​

이렇게 에러를 해결하고, 무사히 데이터를 저장할 수 있게 되었다. 

그 후, 나는 CSS를 뜯어고쳤다.

기존엔 내가 스스로 처음부터 끝까지 작성했는데, React라는 프레임워크를 사용하면서 UI library를 활용하지 않는 다는게 굉장히 비효율적이라고 생각되어서 Ant Design 이라는 React UI Library를 사용했다.

​

Ant Desig을 적용할 때도 우여곡절이 많았다.

컴포넌트 내 attribute가 기존의 html 태그와 다른 부분이 다수 존재해서 

많은 에러를 마주했다 허허 

(여기서 새로 안 사실은 ant design 은 target을 인식하지 못해서 다이렉트로 정의해줘야한다는것)
