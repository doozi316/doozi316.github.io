# Spring Data

​

Spring Data는 데이터에 접근하기 위한 스프링 기반의 프로그래밍 모델이다. 

Spring Data는 데이터베이스 작업에 필요한 많은 코드들을 단축시켜준다.

Spring Data는 몇개의 모듈로 이루어져 있다. 

​

Spring Data JPA 는 JPA 기술로 스프링 프로젝트의 개발을 간소화 시켜주는 역할을 한다.

​

Spring Data를 쓴다면, 어플리케이션 내 각 도메인 엔티티마다 repository interface를 정의해야한다.

repository는 CRUD(Create, Read, Update, Delete) 메소드와 데이터를 정렬, 페이징 기능을 포함한다.

​

``@Repository`` 는 바로 그 기능을 하는 어노테이션이다. (interface 상단에 써줌으로써 근본적인 interface는 repository라는 걸 나타낸다)

repository는 CrudRepository, PagingAndSortingRepository, JpaRepository 등을 통해 extends 해줄 수 있다.

​

​

​

# JpaRepository

​

JpaRepository는 좀 더 구체적이고 확장된 JPA repository이다.

JpaRepository는 CrudRepository와 PagingAndSorgingRepository를 포함하는 포괄적인 API이다.

즉, crud기능과 페이징, 정렬기능까지 갖고 있다는 것.(짱인데?)

​

​

## Spring Boot JpaRepository Example

​

​

아래는 City라는 엔티티와 JpaRepository를 다루는 Spring Boot 예제이다. 

​
![](https://doozi316.github.io/assets/images/spring1/1.png)

<br/>

```
         <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-data-jpa</artifactId>
        </dependency>

        <dependency>
            <groupId>com.h2database</groupId>
            <artifactId>h2</artifactId>
            <scope>runtime</scope>
        </dependency>
jpa를 사용하기 위해선 의존 주입을 해야한다.

위 소스코드에선 jpa와 h2 데이터베이스가 주입된 것을 확인할 수 있다. 

spring-boot-starter-data-jpa는 Spring Data JPA와 Hibernate를 사용하게 해주는 스타터이다.

Hibernate란 자바기반의 ORM(Object Relationship Mapper)이다. 

쉽게 이야기 해서, 자바 객체를 RDMBS의 하나의 ROW로 맵핑해준다. Entity Bean 과 비슷한 기능을 한다.

JPA(Java Persistence API)는 ORM의 표준 스펙이다.

데이터베이스는 h2외에 mysql, oracle 등 아무거나 써줘도 상관 없다.

​

```​
package com.zetcode.model;

import java.util.Objects;
import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;
import javax.persistence.Table;

@Entity //JPA를 사용해서 테이블과 매핑할 클래스에 붙이기
@Table(name = "cities") //엔티티와 매핑할 테이블 지정(name 속성으로 매핑할 테이블 이름을 알린다)
public class City {

    @Id //기본 키 생성
    @GeneratedValue(strategy = GenerationType.AUTO) //기본 키 자동 생성
    private Long id;

    private String name;
    private int population;


    public Long getId() {

        return id;
    }

    public void setId(Long id) {

        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {

        this.name = name;
    }

    public int getPopulation() {

        return population;
    }

    public void setPopulation(int population) {

        this.population = population;
    }
```

<br/>

위 소스코드는 City 엔티티를 정의하고 있다. City는 id, name, population을 애트리뷰트로 갖는다.

​
```
package com.zetcode.repository;

import com.zetcode.model.City;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

@Repository
public interface CityRepository extends JpaRepository<City, Long> {

}
```


CityRepository부분이다. JpaRepository로부터 extends 되고 있다. <> 괄호 부분을 보면,

엔티티와 기본 키의 타입을 제공하는 걸 확인할 수 있다.

​
```
 @Autowired
    private CityRepository cityRepository;
```

<br/>

위 소스코드는 MyRunner.java 부분의 일부를 발췌한 것이다.

Autowired 어노테이션을 통해 CityRepository를 상속해주고있다. 

이런식으로 정의해줌으로써 JpaRepository의 다양한 메소드를 사용할 수 있다.

예를 들면 

```
cityRepository.save(); // insert

cityRepository.count();

cityRepository.findAll();

cityRepository.deleteAllInBatch(); // 내림차순 정렬로 보이기
```
