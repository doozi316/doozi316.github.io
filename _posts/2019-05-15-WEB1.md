## WWW (World Wide Web) 이란?

​



보통 인터넷의 전부라고 생각될 수도 있지만 그렇지 않음

www은 인터넷 기반의 대표 서비스 중 하나이다.


​

물리적인 하나의 컴퓨터에는 여러 개의 서버가 동작 가능하다.

각각의 서버들은 포트라는 값으로 구분돼서 동작한다.

Web은 80번 포트 이용한다.(아래의 표 참고)

​

예) 인터넷 기반의 대표 서비스


| 이름 | 프로토콜 | 포트 | 기능 |
|:---:|:-----:|:---:|:---:|
|  www| HTTP | 80 | 웹 서비스 |
| Email | SMTP/POP3/IMAP | 25/110/114 | 이메일 서비스 |
| FTP | FTP | 21 | 파일 전송 서비스 |
| DNS | TCP/UDP | 53 | 네임 서비스 |
| NEWS | NNTP | 119 | 인터넷 뉴스 서비스 |


<br/>
<br/>

용어정리)

__인터넷__

TCP/IP 기반의 네트워크가 전세계적으로 확대되어 하나로 연결된 네트워크들의 네트워크(네트워크의 결합체)

__프로토콜__

데이터를 주고받기위한 규칙

__포트__

네트워크 서비스나 특정 프로세스를 식별하는 논리 단위 (하단에 IP와 함께 따로 설명해둠)

## HTTP란?

​


간단히 말하자면, 웹에서 서로 통신하기 위한 수단

서버와 클라이언트가 인터넷 상에서 데이터를 주고 받기 위한 프로토콜이다.

​

​

### HTTP 작동방식

​

HTTP는 서버/클라이언트 모델을 따른다

![](https://doozi316.github.io/assets/images/WEB1/1.png)


__장점__

불특정 다수를 대상으로 하는 서비스에 적합.

클라이언트와 서버가 곗고 연결된 형태가 아니기때문에 클라이언트와 서버간의 최대 연결 수 보다 훨씬 많은 요청과 응답을 처리할 수 있다.

​

__단점__

무상태(Stateless) : 서버가 요청에 대한 응답 후 바로 연결을 끊어버리기 때문에, 새로 요청한 클라이언트가 이전의 클라이언트와 같은지 확인 불가

해결 방안 : 정보를 유지하기 위한 Cookie

## URL이란?


​

인터넷 상의 자원의 위치

특정 웹 서버의 특정파일에 접근하기 위한 경로 혹은 주소

접슥프로토콜:// IP 주소 또는 도메인 이름 / 문서의 경로 / 문서 이름

​

쉽게 설명하자면, __IP__ 는 집 주소를 의미하고

__포트__ 는 집 안의 각각의 방을 의미한다. (포트는 각각 다른 번호를 갖고 있어야한다. 0보다 크다)

한 방은 하나의 서버만 갖는다.

#### ❷ request (클라이언트가 서버에게 요청하는 단계)

​

__요청데이터 포맷__ 이라는 규칙으로 요청이 이루어짐

요청 데이터는 __요청 헤더__ 와 __요청 바디__ 로 구성됨

​

__요청헤더 :__

---첫줄-----

+ 요청 메소드 (GET, PUT, POST, PUSH, OPTIONS 등의 요청 방식이 온다.)

+ 요청 URL (요청하는 자원의 위치를 명시),

+ HTTP 프로토콜 버전

---두번째 줄---

+ 헤더 명;

+ 헤더 값;

(각 줄은 라인피드와 캐리지 리턴으로 구분됨)

​

< __요청 메소드__ 의 종류 >

+ GET : 정보를 요청하기 위해서 사용한다. (SELECT) *

+ POST : 정보를 밀어넣기 위해서 사용한다. (INSERT)

+ PUT : 정보를 업데이트하기 위해서 사용한다. (UPDATE)

+ DELETE : 정보를 삭제하기 위해서 사용한다. (DELETE)

+ HEAD : (HTTP)헤더 정보만 요청한다. 해당 자원이 존재하는지 혹은 서버에 문제가 없는지를 확인하기 위해서 사용한다.

+ OPTIONS : 웹서버가 지원하는 메서드의 종류를 요청한다.

+ TRACE : 클라이언트의 요청을 그대로 반환한다. 예컨데 echo 서비스로 서버 상태를 확인하기 위한 목적으로 주로 사용한다.

​

GET 은 요청할 때 가지고 가야할 자원 등을 URL에 기재함으로써 가져감 때문에 요청 바디에 아무것도 기재할 필요 없음

#### ❸ response 서버가 클라이언트에게 응답하는 단계

​

__응답 데이터 포맷__ 이라는 규칙으로 요청이 이루어짐

응답 데이터는 __응답 헤더__ 와 __응답 바디__ 로 구성됨

​

__응답헤더 :__

---첫줄-----

+ HTTP 프로토콜 버전

+ 응답 코드

+ 응답 메시지

---나머지---

+ 날짜

+ 웹 서버 이름, 버전

+ 콘텐츠 타입

+ 캐시 제어방식

+ 컨텐츠 길이 등

​

​

__응답 바디 :__

실제 응답 리소스 데이터
