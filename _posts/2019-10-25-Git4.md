# 특정 폴더나 파일의 commit 막기

​

​

최근 나는 spring boot와 react.js를 연동시킨 프로젝트를 진행 중이다.

초기 셋업을 하다보니 commit 에 올라가지 않았으면 하는 패키지들이 너무 많았고, 이를 막을 방법을 찾아보았다.

​

그 방법은 바로 **.gitigore**을 사용하는 것!

​

```.gitigore``` 에 commit 을 원치 않는 폴더나 파일을 입력해주면, 얘들은 commit이 되지 않는다.

​

​

​
![](https://doozi316.github.io/assets/images/GIT4/1.png)

​

위는 내가 진행중인 프로젝트의 구조이다. 어후 너무 많다 많아...

저 중에 ```.mvn``` , ```.settings``` , ```.vscode``` , target은 commit시킬 때 포함되지 않았으면 한다.

​
![](https://doozi316.github.io/assets/images/GIT4/2.png)

```.gitigore``` 을 열어보니, 이미 많은 파일 및 폴더들이 설정되어 있었다.

​

그중 빠져있는 ```.mvn```  디렉토리도 commit이 막히길 원해서 아래쪽에 한줄 더 추가해줬다.

뒤에 붙어있는 슬래쉬(/)는 하위 디렉토리까지 모두 막히는 걸 의미한다.

​

​

github 원격저장소 확인 결과,

<br/>

![](https://doozi316.github.io/assets/images/GIT4/3.png)

<br/>

원하는 파일 및 폴더들만 올라간 것을 확인할 수 있었다! ^0^

# 원격 저장소의 잘못 업로드된 파일 및 폴더 삭제하기

​

​

만약 기존에 위의 방법을 몰라 원격저장소에 잘못 commit된 파일들이 있다면 

아래의 방법을 따르라.

​

1. git rm -r --cached 삭제할 디렉토리 or 파일명

​

2. git commit -m 'commit message...'

​

3. git push origin master
