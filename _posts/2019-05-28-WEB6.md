## Servlet이란?

<br/>
<br/>



__서블릿__ 은 이클립스 내부에서 설정한 runtime, 즉, 톰캣이라는 WAS로 동작된다.

이후 웹 브라우저를 이용해서 톰캣 서버에 URL 요청을 보내고 실행된다.



__서블릿__ 을 이해하기 위해선 __자바 웹 어플리케이션__ 에대한 개념을 알아야한다.


<br/>
<br/>
<br/>

### 자바 웹 어플리케이션 (Java Web Application)


<br/>
<br/>


- WAS에 설치 (deploy) 되어 동작하는 어플리케이션
<br/>

- 자바 웹 어플리케이션에는 HTML, CSS, 이미지, 자바로 작성된 클래스
(Servlet도 포함됨, package, 인터페이스 등), 각종 설정 파일 등이 포함된다.

<br/>
<br/>



#### 자바 웹 어플리케이션의 폴더 구조

<br/>

![](https://doozi316.github.io/assets/images/WEB6/1.PNG)

<br/>

웹 어플리케이션은 혼자서 동작하는 게 아니라 WAS에 의해서 동작된다.

이 처럼 다른 프로그램의 도움이나 영향을 받는 웹 어플리케이션이나 프레임 워크엔 반드시 지켜야할 약속이 존재한다.

그 지켜야할 약속중 하나가 바로 위 그림과 같은 폴더 구조를 지켜야한다는 것이다.



<br/>
<br/>

- WEB-INF 라는 폴더가 반드시 존재해야한다.
<br/>
- WEB-INF 하위엔 web.xml이라는 파일이 존재하는데, 배포 기술자라고 하며 이 웹 어플리케이션에 대한 정보들을 다 가지고 있다고 생각하면 된다.
하지만 3.0 이상에선 반드시 필요하지는 않게 됐다. (어노테이션이 그것의 역할을 대신했기 때문)
<br/>
- WEB-INF 하위에 또한 lib, classes라는 폴더가 존재할 수 있다.
<br/>
- lib 폴더는 library의 약자로, 각종 자료 파일을 넣을 수 있다.
<br/>

- classes 파일엔 자바 패키지, 서블릿 등과 같은 class 파일이 들어간다.
<br/>

- 그 외의, WEB-INF와 같은 위치에는 이미지나 각종 리소스들이 존재할 수 있다.


<br/>
<br/>
<br/>


### Servlet

<br/>
<br/>


자바 웹 어플리케이션의 구성 요소 중 동적인 처리를 하는 프로그램의 역할을 한다.



- 서블린은 WAS에서 동작하는 JAVA 클래스 이다.
<br/>
- 서블릿은 HttpServlet 클래스를 상속받아야한다.
<br/>
- 서블릿과 JSP로부터 최상의 결과를 얻으려면, 웹 페이지를 개발할 때 이 두가지(JSP, 서블릿)를 조화롭게 사용해야한다.
예) 웹 페이지를 구성하는 화면(HTML)은 JSP로 표현하고, 복잡한 프로그래밍은 서블릿으로 구현
