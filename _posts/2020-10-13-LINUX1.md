# 리눅스 명령어 및 용어 정리

## 명령어

- `$`
    - 일반 유저의 쉘
- `#` 
    - root 유저의 쉘
- `:`
    - ex 모드
    - 라인 명령 모드
    - 화면 맨 아랫줄에서 명령 하는 모드
- `:q` 
    - 편집기 종료
- `:q!`
    - 변경된 내용이 있더라도 저장하지 않고 종료
- `:wq` 
    - 변경된 내용을 저장하고 종료
- `i`, `a`, `o`, `I`, `A`, `O`
    - 입력모드로 전환
    - 편집모드, Input mode, Insert mode
    - 글자를 입력하는 문서 모드
    - 화면 아래에 `--INSERT--` 라고 표시된다.
- `esc`
    - 명령 모드
    - 기본 모드
    - 이동, 수정, 삭제, 복사, 붙이기, 탐색 등을 한다.
- `pwd` 
    - print name of current working directory
    - 현재 디렉토리 위치 보기
    
    ```
    $ pwd
    ```

    - 옵션
        - 없음
- `ls`
    - list segments
    - 현재 디렉토리의 파일 목록을 보는 명령어

    ```
    $ ls [옵션] [경로 혹은 조건]
    ```

    - 옵션
        - `-a` 
            - 숨겨진 파일도 출력
        - `-l`
            - 파일의 상세한 정보 (권한, 용량, 수정날짜 등)을 볼 수 있다.
            - `ls -l` 은 `ll` 명령어와 같다
        - `-lh` 
            - 파일의 용량을 단위와 함께 출력
        - `-S`
            - 크기 순으로 출력
        - `-r`
            - 역순으로 출력
        - `-t`
            - 시간 순으로 출력
        - `-F`
            - 파일과 디렉토리를 구별해서 출력 (디렉토리 뒤에 /가 붙음)
        - `-x`
            - 정렬 순서를 가로로 출력
        - `-u` 
            - 최종적으로 사용된 시간
    - 예)
    
        ```
        ls -tral
        ls *txt -tla // *.txt 파일 리스트 출력
        ```

- `cd`
    - change directory
    - 디렉토리를 변경하는 명령(이동)

    ```
    $ cd [경로]
    ```
    - 옵션
        - 없음
    - 상대경로 사용 방법
        - `cd 경로` : 현 디렉토리의 하위 디렉토리로 이동한다.
        - `cd ./경로` : 현 디렉토리의 하위 디렉토리로 이동한다.
        - `cd ../경로` : `..`은 현 디렉토리보다 한단계 상위 디렉토리를 의미한다.
    - 절대경로 사용 방법
        - `cd /경로` : 경로에 해당하는 위치로 이동
        - `cd ~/경로` : `~`은 자신의 루트 디렉토리를 의미한다.
    - 그 외 사용 방법
        - `cd` : 루트 디렉토리로 이동
        - `cd ~` : 루트 디렉토리로 이동
        - `cd /`: 루트 디렉토리로 이동
        - `cd ~ (아이디)` : 해당 아이디의 홈 디렉토리로 이동
- `tar`
    - 파일을 하나로 모으되 압축하지 않음
    
    ```
    $ tar -c[옵션] [압축파일명] [압축할 파일 혹은 디렉토리] // 압축 할 때
    $ tar -x[옵션] [압축파일명] [압축할 파일 혹은 디렉토리] // 압축 풀 때
    ```

    - 옵션
        - `-c` : 묶음 파일 생성
        - `-x` : 묶음 파일 해제
        - `-v` : 작업 내용 출력
        - `-p` : 권한을 그대로 유지
        - `-z` : gzip 형식으로 압축 혹은 압축해제
        - `-f` : 사용할 파일의 tar지정
    - 예)

        ```
        $ tar -czvf test_tar.tar.gz *txt // TXT 파일을 모두 압축한다
        $ tar -xzvf test_tar.tar.gz //압축을 해제한다.
        ```
    
- `gzip` 
    - 파일을 모을 수 없지만 압축을 담당
    
    ```
    $ gzip [옵션] [압축파일명] // 압축 할 때
    ```
    
    - 옵션
        - `-n` : n은 1부터 9까지 숫자로, 1이 가장 빠르지만 압출율은 가장 낮음
        - `-c` : 압축 결과를 출력하고 원본 파일은 그대로 놔둠
        - `-d` : 압축 해제
        - `-f` : 강제 압축
        - `-l` : 압축 파일의 정보 출력
        - `-r` : 디렉토리를 지정 시 디렉토리에 포함된 모든 파일 압축
        - `-t` : 압축 파일 테스트
        - `-v` : 압축 혹은 해제 시 자세한 정보 출력
        - `-h` : 도움말 출력
        - `-V` : 버전 정보 출력
    - 예)

        ```
        $ gzip -9v test_zip.txt : 압축하기
        $ gzip -d test_zip.txt.gz : 압축 해제
        ```

- `gunzip`
    
    ```
    $ gunzip [옵션] [압축파일명]
    ```

    - 옵션
        - `-l` : 압축 파일 정보 출력
        - `-r` : 디렉토리를 지정 시 디렉토리 내의 모든 파일의 압축 해제
        - `-t` : 압축 파일 테스트
        - `-v` : 압축해제 시 자세한 정보 출력
        - `-h` : 도움말 출력
        - `-v` : 버전 정보 출력
    - 예)

        ```
        $ gunzip -v test_gunzip.txt.gz // 압축하기
        ```

- `cp`
    - copy
    - 파일 및 디렉토리를 복사

    ```
    $ cp [옵션] [원본] [복사본]
    ```

    - 옵션
        - `-a` : 원본 파일의 속성, 링크 정보를 그대로 유지
        - `-b` : 같은 이름의 파일이 있을 때 백업 파일 생성 후 복사
        - `-f` : 같은 이름의 파일이 있을 때 삭제 후 복사 (덮어쓰기)
        - `-u` : 같은 이름의 파일이 있을 때 변경날짜가 같거나 최근이면 복사하지 않음 (업데이트)
        - `-i` : 같은 이름의 파일이 있으면 사용자에게 물어본다
        - `-r` : 하위 디렉토리까지 복사
        - `-p` : 권한까지 복사
        - `-v` : 복사 과정을 보여준다
    - 예)

        ```
        $ cp -r test test1 // test의 하위 디렉토리를 포함해서 test1로 복사하라
        $ cp a0? test // a0? 조건을 만족하는 파일을 test폴더로 복사하라
        $ cp -i a02 b01 : a02 파일을 b01에 복사하되, 중복 파일을 물어보라
        ```

- `ps`
    - process
    - 작동중인 프로세스를 나타낸다

    ```
    $ ps [옵션]
    ```

    - 옵션
        - `e` : 모든 프로세스를 표시
        - `f` : 전체 경로로 프로세스를 표시
        - `l` : 긴 포맷으로 출력
        - `u` : 실행한 유저 및 실행 시간을 표시
        - `a` : 다른 유저의 프로세스도 표시
        - `x` : 터미널 제어 없이 프로세스 현황 보기
- `mkdir`
    - make directory
    - 디렉토리를 만듬
    
    ```
    $ mkdir [옵션] [디렉토리 명]
    ```
    
    - 옵션
        - `-e` : 암호화 상속으로 디렉토리를 작성
        - `-m` : (Mode) 새로 작성된 디렉토리에 대한 사용 권한 비트를 Mode 변수가 지정하는 값으로 설정
        - `-p` : 누락된 중간 경로 이름 디렉토리를 작성
    - 예)

        ```
        $ mkdir Test // 현재 작업 디렉토리에 Test 라는 새 디렉토리를 작성
        ```
- `grep`
    - 파일 내용을 검색하여, 해당하는 단어나 문자를 찾는다.

    ``` 
    $ grep [옵션] [단어] [파일명]
    ```
    
    - 옵션
        - `-c` : 파일이 아닌 총 숫자를 출력
        - `-C 숫자` : 일치하는 행에서 위와 아래로 지정한 숫자만큼의 행을 추가로 더 보여준다
        - `-A 숫자` : 일치하는 행에서 아래로 지정한 숫자만큼의 행을 추가로 더 보여준다
        - `-i` : 대소문자를 구분하지 않는다.
        - `-n` : 파일 내 행 번호를 함께 출력한다.
        - `-v` : 지정한 문구와 일치하지 않는 것들을 보여준다.
        - `-r` : 하위 디렉토리까지 검색
        - `-b` : 블록 번호 표시
    - 예)

        ```
        $ grep -nC 1 scv star.txt
        ```
- `rm`
    - remove
    - 파일이나 디렉토리 삭제

    ``` 
    $ rm [옵션] [파일/디렉토리]
    ```

    - 옵션
        - `-f`, `--force` : ;삭제 여부를 묻지 않고 강제 삭제를 실행
        - `-i` : 삭제 시 일일이 삭제할 것인지 물어본다.
        - `-d`, `--dir` : 디렉토리를 삭제. 디렉토리 안에 파일이나 서브 디렉토리가 있을 경우 삭제되지 않는다.
        - `-r`, `-R`, `--recursive` : 하위 디렉토리를 포함한 모든 파일들을 삭제한다.
        - `-v`, `-verbose` : 삭제 과정을 출력한다
        - `--help` : 해당 명령어의 도움말을 보여주고 실행을 종료한다.
        - `--version` : version 정보를 출력하고 실행 종료
- `cat`
    - 파일을 확인하는데 사용한다
    
    ```
    $ cat [옵션] [> 혹은 >>] [파일명]
    ```

    - 옵션
        - `-n` : 행 번호를 출력한다
        - `-b` : 빈 행에는 번호를 출력하지 않는다.
        - `-e` : 각 행 끝에 $를 단다
        - `-s` : 연속 되는 빈 행을 하나로 출력한다.
    - 예)

        ```
        $ cat -n test_file.txt // test_file.txt 파일을 행 번호와 함께 출력
        $ cat -nbes test_file.txt // test_tile.txt // 파일을 행번호와 함께 출력하되, 빈 행의 번호는 빼고, 연속되는 빈 행은 하나로 출력. 그리고 각 행의 끝에는 $ 출력
        ```

- `scp`
    - secure copy(remote file copy program)
    - `ssh` 를 이용해 네트워크로 연결된 호스트 간에 파일을 주고 받는 명령어
    - 로컬 -> 리모트(보내기)
    - 리모트 -> 로컬(가져오기)
    - 리모트 -> 리모트(다른 호스트끼리 전송)
    - `ssh`를 이용하기 때문에 password를 입력하거나 `ssh` 키 파일과 같은 identity file을 이용해 파일 송수신이 가능하다.
    
    ```
    $ scp [옵션] [소스] [타겟]
    $ scp 목적파일명(경로) 유저명@IP 주소:목적디렉토리
    $ scp 유저명@IP주소:파일디렉토리 목적파일명(경로)
    $ scp 유저명@IP주소:파일디렉토리 유저명@IP주소:파일디렉토리
    ```

    - 옵션
        - `-r` : 재귀적으로 모든 폴더 복사. 폴더를 복사할 때 사용하는 옵션으로 이때 전송하고자 하는 대상은 폴더로 지정.
        - `-P` : `ssh` 포트를 지정
        - `-i` : `ssh` 키 파일과 같은 identity file의 경로를 지정하는 옵션
        - `-v` : verbose 모드로 상세 내용을 보며 디버깅할 때 사용
        - `-p` : 파일의 수정시간과 권한을 유지


- `|`
    - pipe
    - 프로그램 간 결과를 전송하는 명령어
    - 두 명령어를 복합해서 명령을 내릴 수 있다.
    - 예)

        ```
        $ cat -n star.txt | tail -2 // 앞의 cat 명령어대로 줄번호를 적은 파일을 출력할 것을 다시 입력으로 받아 tail의 명령어를 수행하여 뒤에서부터 2번째 행까지 나타낸다.
        ```

- `>`
    - redirection
    - 파일 간의 결과 전송을 의미
    - 프로그램을 돌려 나온 결과를 화면에 출력하는 것이 아니라 리다이렉션 다음에 나오는 파일명에 저장을 해달라는 것을 의미
    - 어펜드(append `>>`)는 리다이렉션의 파일 뒤에 붙이기 기능이 추가된 것
    - 리다이렉션은 기존 같은 이름의 파일이 있을 경우, 실행되지 않거나 혹은 지우고 새 파일을 만든다.
    - 어펜드는 기존 파일 뒤편에 합병해준다.
    - 예)

        ```
        $ cat TEST_FILE_ORIGIN.txt > TEST_FILE_TARGET.txt // TEST_FILE_ORIGIN.txt를 TEST_FILE_TARGET.txt에 덮어씌움
        $ cat TEST_FILE_ORIGIN.txt >> TEST_FILE_TARGET.txt // TEST_FILE_ORIGIN.txt 파일을 TEST_FILE_TARGET.txt 파일에 합병
        $ ls *txt > TEST_FILE_ORIGIN.txt // txt로 끝나는 파일을 출력하는 대신, TEST_FILE_ORIGIN.txt라는 파일을 만들고 그 안에 저장
        ```

### 압축 확장자 정리

- `zip` 
    - 윈도우 환경에서 가장 일반적으로 사용되는 압축 확장자
    - 거의 모든 OS 환경에서 호환된다
    - 용량 압축이 낮다
    - 예)

        ```
        $ zip -r FILENAME.zip FILENAME // 압축하기
        $ unzip FILENAME.zip // 압축 풀기
        ```

- `tar`
    - 리눅스 환경에서 일반적으로 사용되고, 사실상 압축파일이 아닌 여러개 파일을 하나로 묶는 개념
    - 압축에 소비되는 시간, CPU가 적다
    - 용량 압축이 거의 없다
    - 예)

        ```
        $ tar -cf FILENAME.tar FILENAME // 압축하기
        $ tar -xf FILENAME.tar // 압축풀기
        $ tar xvf FILENAME.tar -C /home/ // 원하는 위치에 압축 풀기
        ```
- `tar.gz`
    - 리눅스 환경에서 대부분 사용하고 있고, 위에 합쳐진 TAR 파일을 압축하는 방ㅅ기으로 가장 좋은 압축옵션
    - 용량 압축이 높지만 CPU는 많이 소비되지 않는다.
    - TAR.BZ2 보다는 용량 압축이 낮다
    - 예)

        ```
        $ tar -zcvf FILENAME.tar.gz FILENAME // 압축하기
        $ tar -zxvf FILENAME.tar.gz // 압축풀기
        $ tar xvf FILANAME.tar.gz -C /home/ // 원하는 위치에 압축 풀기
        ```

## 용어

- SSH
    - Secure Shell Protocol
    - 네트워크 프로토콜 중 하나
    - 컴퓨터와 컴퓨터가 인터넷과 같은 Public Network를 통해 서로 통신할 때 보안적으로 안전하게 통신하기 위해 사용하는 프로토콜
    - 데이터 전송, 원격 제어에서 주로 사용된다.
    - 데이터 전송 예) 원격 저장소인 깃헙에 소스코드를 푸쉬할 때 SSH를 이용해 파일을 전송하게 된다.
    - 원격제어 예) AWS의 인스턴스 서버에 접속하여 해당 머신에 명령을 내리기 위해 SSH를 통한 접속을 해야한다.
    - FTP나 Telnet과 같은 다른 컴퓨터와 통신을 위해 사용되는 프로토콜에 비해 보안이 우수하다.
        - `Private Key` and `Public Key`
            - SSH는 다른 컴퓨터와 통신을 하기 위해 접속할 때 일반적으로 사용하는 비밀번호의 입력을 통한 접속을 하지 않는다.
            - SSH는 한 쌍은 Key를 통해 접속하려는 컴퓨터와 인증 과정을 거치게 된다.(`Private Key`, `Public Key`)
            - `Public Key`
                - 공개키
                - 공개되어도 비교적 안전한 Key
                - `Public Key`를 통해 메시지를 전송하기 전 암호화를 하게 된다.
                - `Public Key`로는 암호화는 가능하지만 복호화는 불가능
            - `Private Key`
                - 외부에 노출되어선 안되는 Key
                - 본인의 컴퓨터 내부에 저장하게 되어 있다.
                - `Private Key`를 통해 암호화된 메시지를 복호화 할 수 있다.
            - `Public Key`를 통신하고자 하는 컴퓨터에 복사하여 저장
            - 요청을 보내는 클라이언트 사이드 컴퓨터에서 접속을 요청할 때 응답을 하는 서버 사이드 컴퓨터에 복사되어 저장된 `Public Key` 와 클라이언트 사이드에 해당 `Public Key` 와 쌍을 이루는 `Private Key`와 비교를 하여 서로 한 쌍의 Key인지 아닌지를 검사한다. 
- SFTP
    - ftp와 유사한 사용자 인터페이스가 있는 파일 전송 프로그램
    - SSH FTP(File Transfer Protocol) 를 사용하여 서버에 대한 보안 연결을 만든다
    - ftp 명령으로 사용할 수 있는 옵션 중 일부는 sftp 명령에 포함되어 있지 않지만 대부분의 명령이 포함되어 있다.

    ```
    $ sftp 계정이름@ip
    $ sftp> get [파일이름] // 파일 다운로드
    $ sftp> get /home/a.png
    $ sftp> put [파일이름] // 파일 업로드
    ```

    - 그 외 명령어
        - `mget` : 다수의 파일을 다운로드
        - `mput` : 다수의 파일을 업로드
        - `ls` : 접속한 sftp의 파일 목록 출력
        - `!ls` : 로컬 서버의 파일 목록 출력
        - `!명령어` : 로컬 서버에서 실행
